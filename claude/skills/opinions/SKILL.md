# Opinions and viewpoints

When working on something that would benefit from Jessenia's viewpoints, preferences, judgment, taste, or personal philosophy, first check whether `~/OPINIONS.md` is available.

If `~/OPINIONS.md` is available, read it before making opinion-sensitive decisions.
If it is not available, ask Jessenia for her viewpoint before deciding.

Use `~/OPINIONS.md` as a compact source of durable personal judgment, not a dumping ground for every preference.
Keep it small, high-signal, and easy to scan.

Do not silently add to, restructure, or rewrite `~/OPINIONS.md`.
When adding an opinion, ask Jessenia whether it should be merged into an existing line, replace an existing line, or be added as a new line.
Prefer merging related opinions over creating duplicates.
Do not preserve weak, stale, or redundant opinions just because they already exist.
